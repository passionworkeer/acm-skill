# 竞赛常用模式

> 这些模式不是具体算法，而是竞赛中高频出现的通用套路。每个模式都有明确的"何时用"判断标准。

---

## 1. 倍增法（Binary Lifting）

**何时用**：需要快速跳过多个相同规模的步骤，或在有根树上快速向上跳。

**核心思想**：预处理 `f[i][j]` = 从节点 i 向上跳 2^j 步到达的节点，查询时从大到小贪心跳。

**典型用法**：LCA（最近公共祖先）、K 祖先节点、树上距离。稀疏表本质也是倍增——按 2 的幂次合并区间。

```cpp
// 预处理：O(n log n)
for (int j = 1; j < LOG; j++)
    for (int i = 1; i <= n; i++)
        anc[i][j] = anc[anc[i][j-1]][j-1];

// 查询 K 祖先
int kthAncestor(int u, int k) {
    for (int j = 0; j < LOG; j++)
        if (k & (1<<j)) u = anc[u][j];
    return u;
}
```

**判定标准**：问题的合并具有结合律（跳两步等于跳一次再跳一次），且答案可以二进制拆分。

## 2. CDQ 分治

**何时用**：三维或更高维的偏序统计，且一维可以排序。

**核心思想**：按 x 排序后递归处理左右两部分，用归并/树状数组统计跨越中线的贡献，避免 O(n²)。

**典型用法**：三维偏序（CDQ(x) + 归并(y) + BIT(z)）、CDQ 优化 DP（将 DP 转移视为三维偏序）。所有满足偏序单调性的高维 DP 转移均可尝试。

```cpp
void cdq(int l, int r) {
    if (l == r) return;
    int mid = (l+r)>>1;
    cdq(l, mid); cdq(mid+1, r);
    // 处理跨越 [l,mid] 和 [mid+1,r] 的贡献
    // 通常用归并按第二维排序，再用 BIT 按第三维累加
    merge_and_process(l, mid, mid+1, r);
}
```

**判定标准**：DP 转移形如 `dp[i] = max(dp[j] + w(j,i))`，其中 j < i，且 w 满足可合并性。

## 3. 离线 + 逆序

**何时用**：问题包含"加边/删边"或"插入/删除"操作，且操作可逆。

**核心思想**：将所有操作按时间顺序离线，逆序处理时"删除"变成"插入"，利用可逆数据结构（支持 undo 的 DSU）完成。

**典型用法**：动态连通性（按时间加边，用可撤销 DSU 离线回答）；从右往左扫描避免删除；线段树分治（把每条边的有效时间区间挂到线段树节点上）。

```cpp
// 可撤销并查集
stack<pair<int*, int>> stk;  // 地址 + 旧值
void undo() {
    while (!stk.empty()) {
        *stk.top().first = stk.top().second;
        stk.pop();
    }
}
```

**判定标准**：所有操作均可逆（或可以离线转为逆序处理），且需要回答多个时间点的查询。

## 4. 二分答案

**何时用**：最大化最小值、最小化最大值、存在性判定具有单调性。

**核心思想**：把最优化问题转为判定问题——给定阈值 X，检验"是否存在可行解 ≥ X"。判定函数通常是一个多项式算法。

**典型用法**：最小生成树的最大边最小 → 二分最大边；最大平均密度子图；二分图匹配加边验证；最大化第 k 大值。

```cpp
int lo = 0, hi = INF;
while (lo < hi) {
    int mid = (lo + hi + 1) >> 1;
    if (feasible(mid)) lo = mid;   // mid 可行，尝试更大
    else hi = mid - 1;             // mid 不可行，缩小
}
```

**判定标准**：答案具有单调性（可行 / 不可行分界清晰），且检验一个值是否可行比直接求最优简单得多。

## 5. 01 字典树（异或 Trie）

**何时用**：需要在一组数中找与给定数异或最大/最小的值。

**核心思想**：将每个数的二进制位从高位到低位插入字典树，贪心选择与目标位相反的分支使得异或值最大。

**典型用法**：最大异或和子集；第 K 小异或和；子数组异或最大值（边插入边查询）。

```cpp
int queryMaxXor(int x) {
    int cur = 0;
    for (int i = MAXBIT; i >= 0; i--) {
        int b = (x>>i) & 1;
        if (tr[cur][b^1]) cur = tr[cur][b^1];
        else cur = tr[cur][b];
    }
    return x ^ val[cur];
}
```

**判定标准**：问题涉及异或最值或第 K 极值，且数据规模适合 Trie 结构。

## 6. 子集枚举

**何时用**：状态压缩 DP、容斥原理、子集求和。

**核心思想**：`for (int sub = mask; sub; sub = (sub-1) & mask)` 按降序遍历 mask 的所有非空子集，利用 `sub-1` 把最高位 1 变成 0 并将后面位全置 1 的性质恰好枚举所有子集。

**典型用法**：子集 DP 的 `dp[mask] = min(dp[mask], dp[mask\sub] + w[sub])`；容斥原理枚举子集计算补集贡献；SOS DP（子集 DP）在 O(n·2^n) 内求每个 mask 所有子集的最值/和。

```cpp
// SOS DP: f[mask] = sum_{sub ⊆ mask} a[sub]
for (int i = 0; i < n; i++)
    for (int mask = 0; mask < (1<<n); mask++)
        if (mask & (1<<i))
            f[mask] += f[mask ^ (1<<i)];
```

**判定标准**：状态空间为 2^n（n ≤ 20~25），或需要枚举子集计算贡献。

## 7. 分数规划

**何时用**：最大化 / 最小化比值问题，如最大平均密度、最优比率生成树。

**核心思想**：`max (a/b)` → 二分 λ，检查 `max (a - λ·b) ≥ 0`。将除法去掉，通过二分斜率把比值问题转为可行性判定。

**典型用法**：最大平均密度子图；最优比率生成树（Kruskal 比较 a - λb）；比率背包问题。

```cpp
double check(double lambda) {
    double max_val = -INF;
    for (edge e : edges) max_val = max(max_val, e.a - lambda * e.b);
    return max_val >= 0;  // λ 可以更大
}
```

**判定标准**：目标函数是两个线性/可计算值的比值，且每个元素的 a 和 b 可独立计算。

## 8. 莫队（Mo's Algorithm）

**何时用**：离线区间询问，询问之间相互独立，支持 add / remove 为 O(1) 或 O(log n)。

**核心思想**：将 q 个询问按左端点分块排序（块大小 ≈ √n），同一块内按右端点排序，使得左右指针移动次数均摊 O(√n)，总复杂度 O((n+q)√n)。

**典型用法**：静态区间众数、区间不同数数量、区间 mex 等需要离线无法用线段树快速回答的询问。

```cpp
sort(queries.begin(), queries.end(), [](Query a, Query b){
    if (a.l/block != b.l/block) return a.l < b.l;
    return (a.l/block & 1) ? a.r < b.r : a.r > b.r;  // 奇偶优化
});
```

**判定标准**：询问完全离线；区间可合并（add/remove）；无在线更新需求；n, q ≤ 5×10⁴（否则考虑带修莫队或主席树）。

## 9. 质心分解（Centroid Decomposition）

**何时用**：树上路径统计问题，需要 O(n log n) 而非 O(n²) 枚举所有点对。

**核心思想**：递归删除当前连通块的质心，对经过质心的路径单独统计（从质心出发向下维护到各子树的距离），避免重复计算。

**典型用法**：树上距离等于 K 的点对数；带约束的树上路径计数；路径权值最值问题。

```cpp
int compute(int u, int p, int d, vector<int>& dist) {
    dist.push_back(d);
    for (auto [v,w] : g[u])
        if (v != p && !removed[v])
            compute(v, u, d+w, dist);
}
```

**判定标准**：需要在树上对"经过某些点的路径"进行计数或极值查询，且树无明显性质（如链、树dp 天然可做）可供利用。

## 10. 虚树（Virtual Tree）

**何时用**：给定一颗大树和少量关键点，要求在关键点构成的子图上高效查询。

**核心思想**：把所有关键点按 DFS 序排序，相邻关键点两两求 LCA，将这些节点（含原关键点）构成一棵只包含相关节点的树。在虚树上 DP 复杂度为 O(k log k)，而非 O(n)。

**典型用法**：关键点子树聚合；多关键点路径统计；虚树 + 树形 DP。

```cpp
// 关键：按 DFS 序建树
sort(key_nodes.begin(), key_nodes.end(), [](int a,int b){
    return tin[a] < tin[b];
});
stack<int> st; st.push(root);
for (int v : key_nodes) {
    int l = lca(v, st.top());
    while (depth[st.top()] > depth[l]) {
        if (depth[st.second()] >= depth[l]) { /* link */ }
        st.pop();
    }
    st.push(l); st.push(v);
}
```

**判定标准**：树规模大（10⁵+），但每次只涉及 O(k) 个关键点（k ≤ 10⁴），且需要树上路径 / 子树聚合信息。

## 模式选择指南

| 问题特征 | 首先考虑 |
|---------|---------|
| 有单调性（递增/递减）| 单调队列 / 双指针 |
| 可离线、逆操作 | 逆序处理 / 可撤销 DSU |
| 二分可行（max-min 型）| 二分答案 |
| 高维 DP、转移有偏序 | CDQ 分治 |
| 需要历史版本 | 可持久化线段树 |
| 在线 + 区间 + 简单合并 | 线段树 |
| 离线 + 区间 + 复杂合并 | 莫队 |
| 路径/子树询问（单次）| 树链剖分 |
| 树上路径统计（多次）| 质心分解 |
| 关键点树操作 | 虚树 |
| 异或/AND/OR 最值 | 二进制 Trie |
| 递推 + 大指数 | 矩阵乘法 / 快速幂 |
| 分数比值优化 | 分数规划 |
| n ≤ 2ⁿ 状态枚举 | 子集枚举 / SOS DP |
