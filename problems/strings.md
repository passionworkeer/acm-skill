# 字符串经典题解

> 题目特征 → 算法选择 → 核心思路

---

## 1. 匹配类

### 精确匹配（单模式）
**题目特征**：在文本 T 中找模式串 P 是否出现
**算法**：KMP
**模板**：`kmp.hpp`
**核心**：`next[]` 表（π 函数），匹配时若失配跳转到 `next[j]`

### 多模式匹配
**题目特征**：在文本中同时查找多个模式串的出现
**算法**：AC 自动机
**模板**：`ac-automaton.hpp`
**核心**：Trie 建 fail 指针，文本在 AC 图上跑，fail 树可统计所有模式出现次数

### 字符串哈希
**题目特征**：快速判断子串相等 / 寻找最长公共前缀
**模板**：`polyhash.hpp`
**核心**：双模数哈希（避免生日攻击），LCP 用二分 + 哈希验证

---

## 2. 回文类

### 最长回文子串
**算法**：Manacher（线性）
**模板**：`manacher.hpp`
**核心**：维护臂长和右端点，`O(n)` 求所有以 i 为中心的最长回文半径

### 本质不同回文串数量 / 出现次数
**算法**：回文树（Eertree）
**模板**：`palindromic-tree.hpp`
**核心**：每个新字符在树上新增或跳 fail 到已有节点；`cnt` 统计出现次数（需两次遍历）

### 回文子串构造 / Lyndon
**模板**：`lyndon.hpp`
**核心**：Lyndon 串 = 字典序小于所有后缀；Duval 算法求 Lyndon 分解

---

## 3. 后缀结构

### 后缀数组（SA）
**模板**：`suffix-array.hpp`（倍增 O(n log n)）或 `suffix-array-linear.hpp`（SA-IS O(n)）
**核心**：`sa[i]` = 排名第 i 的后缀起始位置；`rank[i]` = 以 i 开头后缀的排名
**应用**：Height 数组 → LCP → 两子串最长公共前缀

### 后缀自动机（SAM）
**模板**：`suffix-automaton.hpp`
**核心**：SAM 的每个状态代表所有终止位置集合，`len` = 最长长度，`link` = 最短后缀
**应用**：子串查询、最长公共子串（两串 SAM 交集）、不同子串数量 `Σ(len[i]-len[link[i]])`

### 扩展后缀自动机（多串）
**模板**：`ex-sam.hpp`
**核心**：Trie + SAM，适合判断某串在哪些串中出现

### 后缀数组 RMQ 加速 LCP
**模板**：`suffix-array.hpp` + `sparse-table.hpp`
**核心**：`RMQ(l,r)` = Height[rank[l]+1..r] 的最小值

---

## 4. 字典树

### 前缀匹配 / 统计
**模板**：`trie.hpp`
**核心**：插入 O(len)，每个节点维护 `cnt`（经过次数）、`exist`（是否终止）
**应用**：前缀和、异或和最值（Trie 上贪心）

---

## 5. Z 函数

### 字符串匹配（类似 KMP）
**模板**：`z-function.hpp`
**核心**：`z[i]` = T 与 T[i..] 的最长公共前缀；匹配时拼 S+'#'+T，求 S 的 z 值

---

## 6. 常用套路

| 场景 | 套路 |
|------|------|
| 判断两子串相等 | 字符串哈希 or SA+RMQ |
| 多模式出现次数 | AC 自动机 + fail 树统计 |
| 回文串数量/出现 | 回文树 |
| 子串查询 | SAM（所有子串集合）|
| 前缀统计 | 字典树 |
| 循环同构 | 最小表示法（Lyndon）或字符串哈希 |
| LCP 查询 | SA + Height + RMQ |
